# RK3568 边缘设备部署与内存优化指南

本指南旨在解决在资源受限（如 2GB/4GB RAM）的 ARM64 设备上，运行高负载 Python 应用（如 AI 推理）时的部署与稳定性问题。

------

## 一、 Miniforge 环境部署 (ARM64)

Miniforge 是 ARM 设备上管理 Conda 环境的最佳选择，它默认使用 `conda-forge` 源，对 aarch64 的数学库支持更友好。

### 1. 安装 Miniforge

使用国内镜像源以加快下载速度：

Bash

```
# 下载安装脚本（推荐北京大学镜像）
wget https://mirrors.tuna.tsinghua.edu.cn/github-release/conda-forge/miniforge/LatestRelease/Miniforge3-Linux-aarch64.sh

# window x86使用
curl.exe -L -O https://mirrors.tuna.tsinghua.edu.cn/github-release/conda-forge/miniforge/LatestRelease/Miniforge3-25.11.0-1-Windows-x86_64.exe

# 运行安装程序
chmod +x Miniforge3-Linux-aarch64.sh
./Miniforge3-Linux-aarch64.sh
# 提示中一路输入 yes 即可，安装完成后需重启终端
```

### 2. 配置国内镜像源（可选）

替换 Conda 和 Pip 的源，大幅提升后续环境搭建速度：

Bash

```
# Conda 换源 (清华镜像)
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge/
conda config --set show_channel_urls yes

# 创建并激活环境
conda create -n dhlr_env python=3.11 -y
conda activate dhlr_env

# Pip 换源 (阿里云)
pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/
```

## 二、 OOM (内存溢出) 解决方案

RK3568 运行 AI 推理初始化时常因瞬时内存需求过大被系统 `Killed`。通过以下三个维度的优化可彻底解决。

### 1. 系统级：开启虚拟内存 (Swap)

这是最简单、最有效的临时扩容手段，建议在 2GB 内存的设备上至少开启 **1GB** 的 Swap。

Bash

```
# 1. 创建 1GB 的 Swap 文件
sudo fallocate -l 1G /swapfile

# 2. 设置权限
sudo chmod 600 /swapfile

# 3. 格式化为交换分区
sudo mkswap /swapfile

# 4. 启用交换分区
sudo swapon /swapfile

# 5. (可选) 永久生效：在 /etc/fstab 末尾添加
# /swapfile swap swap defaults 0 0
```

### 3. 应用级：预合成策略

若实时推理依然无法满足性能要求，应采用“**空间换时间**”策略：

- **离线批量合成**：在部署阶段将常用话术（如“系统启动”、“检测异常”）预先合成 `.wav`。
- **调用系统播放器**：使用 `aplay` 等轻量级 C 语言工具播放，避免 Python 进程在播放期间持续占用大量内存。
