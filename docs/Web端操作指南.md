## 系统概述

Web管理界面采用现代化的移动端优先设计，支持触屏操作，主要用于：
- 实时监控灶台状态
- 配置和管理摄像头
- 设置监测区域（ROI）
- 进行设备巡检和演示
- 系统参数配置

### 访问地址

- 前端界面：`http://<设备IP>:8000`

### 底部导航栏

页面底部固定显示导航栏，包含以下六个功能入口：

| 图标 | 名称 | 说明 |
|------|------|------|
| 📊 | 仪表盘 | 实时监控页面（首页） |
| 📷 | 摄像头 | 摄像头设备管理 |
| 🔥 | 灶台 | 灶台区域配置 |
| 🔍 | 巡检 | 设备巡检功能 |
| ⚙️ | 设置 | 系统参数设置 |
| 📋 | 日志 | 系统日志查看 |

## 实时监控（仪表盘）

仪表盘是系统的核心监控页面，提供所有灶台的实时状态概览。

### 页面布局

#### 1. 顶部标题栏

- **标题**：显示"实时监控"及在线状态指示灯（绿色闪烁点表示系统在线）
- **刷新按钮**：手动刷新当前状态
- **全屏按钮**：进入/退出全屏模式，便于大屏幕监控

#### 2. 指示灯状态栏

显示系统三类指示灯的聚合状态：

| 指示灯 | 颜色 | 说明 |
|--------|------|------|
| 动火 | 🟠 橙色 | 任一灶台检测到动火时亮起 |
| 离人 | 🟡 黄色 | 任一灶台检测到无人时亮起 |
| 报警 | 🔴 红色（闪烁） | 任一灶台进入预警/报警/切电状态时亮起 |

#### 3. 灶台状态卡片

每个启用的灶台显示为一张卡片，包含：

##### 基本信息
- **灶台名称**：如"1号灶台"
- **灶台ID**：系统内部唯一标识

##### 状态图标
- **火焰图标** 🔥：橙色=动火中，灰色=未动火
- **人员图标** 👤：绿色=有人值守，灰色=无人

##### 状态文字与倒计时

| 状态 | 显示内容 | 说明 |
|------|----------|------|
| `idle` | 设备空闲 | 灶台未动火 |
| `active_present` | 正常监护中 | 动火中且有人值守 |
| `active_no_person` | XX秒剩余（黄色） | 动火中无人，进入预警倒计时 |
| `warning` | XX秒剩余（黄色） | 预警状态，进入报警倒计时 |
| `alarm` | XX秒剩余（红色发光） | 报警状态，进入切电倒计时 |
| `cutoff` | 已切断电源 | 已触发切电操作 |
| `temp_alarm` | 温度报警 | 温度超过阈值触发的报警 |

##### 底部数据
- **电流值**：显示当前检测到的电流值（单位：A）
- **温度值**：如绑定温度传感器则显示当前温度（单位：°C）

#### 4. 性能监控面板

实时显示系统资源使用情况：

| 指标 | 说明 |
|------|------|
| FPS | 检测帧率（带趋势图） |
| CPU | CPU使用率（带趋势图） |
| NPU | 神经网络处理器负载（带趋势图） |
| 推理延迟 | 单次AI推理耗时（毫秒） |
| 内存使用 | 当前内存占用（MB） |
| 引擎 | 当前推理引擎类型（pytorch/rknn） |

### 卡片状态样式

- **正常状态**：标准玻璃态卡片
- **预警状态**：边框变为黄色发光
- **报警/切电状态**：边框变为红色发光，背景闪烁
- **温度报警**：边框变为橙色发光

## 摄像头管理

用于管理系统中的摄像头设备。

### 摄像头列表

每个摄像头卡片显示：
- **摄像头名称**
- **状态指示**：在线（绿色）/ 离线（红色）
- **类型信息**：USB摄像头 / IP摄像头(RTSP)
- **摄像头ID**

### 操作按钮

| 按钮 | 图标 | 功能 |
|------|------|------|
| 预览 | 👁️ | 打开实时预览窗口 |
| 编辑 | ⚙️ | 修改摄像头配置 |
| 删除 | 🗑️ | 删除摄像头（需确认） |

### 添加摄像头

点击右下角的蓝色 **+** 按钮打开添加对话框。

#### USB摄像头配置

| 字段 | 说明 |
|------|------|
| 名称 | 摄像头显示名称，如"厨房摄像头1" |
| 类型 | 选择"USB摄像头" |
| 选择设备 | 系统自动扫描可用USB摄像头供选择 |

#### IP摄像头(RTSP)配置

| 字段 | 说明 |
|------|------|
| 名称 | 摄像头显示名称 |
| 类型 | 选择"IP摄像头(RTSP)" |
| RTSP地址 | 完整RTSP URL，如 `rtsp://192.168.1.100:554/live/main` |
| 用户名 | RTSP认证用户名（可选） |
| 密码 | RTSP认证密码（可选） |

### 摄像头预览

点击预览按钮后，弹出模态窗口显示摄像头实时画面，自动每秒刷新。

## 灶台配置

用于配置需要监测的灶台区域。

### 灶台列表

每个灶台卡片显示：
- **灶台名称**
- **启用开关**：ON/OFF 控制是否参与监测
- **关联摄像头ID**
- **电流阈值**：判定动火的电流值阈值

### 操作按钮

| 按钮 | 图标 | 功能 |
|------|------|------|
| 编辑 | ✏️ | 修改灶台配置及ROI区域 |
| 删除 | 🗑️ | 删除灶台（需确认） |

### 添加灶台

点击右下角 **+** 按钮打开添加对话框。

| 字段 | 必填 | 说明 |
|------|------|------|
| 名称 | ✅ | 灶台显示名称，如"1号灶台"（用于语音播报） |
| 关联摄像头 | ✅ | 选择监控该灶台的摄像头 |
| 串口分区索引 | ❌ | 对应硬件接线顺序，从0开始（0对应地址0x01） |
| 动火电流阈值 | ❌ | 电流超过此值判定为动火（145表示1.45A） |
| 启用温度传感器 | ❌ | 是否绑定温度传感器 |

### 编辑灶台

#### 基本配置

与添加灶台相同的配置项，另外：
- **名称修改提醒**：如果修改了名称，系统会提示将重新合成语音文件
- **温度传感器地址**：如已绑定则显示当前传感器地址

#### ROI区域编辑

ROI（Region of Interest）是指定灶台在画面中的监测区域。

##### 操作方法

1. **添加顶点**：在摄像头画面上点击/触摸添加区域顶点
2. **拖动调整**：点击已有顶点可拖动调整位置
3. **撤销**：点击"撤销"按钮删除最后一个顶点
4. **清除**：点击"清除区域"按钮删除所有顶点

##### 显示说明

- 绿色多边形区域表示当前设定的监测范围
- 每个顶点显示序号（1、2、3...）
- 选中的顶点会变为蓝色并放大
- 至少需要3个顶点才能形成有效区域

## 设备巡检

设备巡检功能用于验证系统各项功能是否正常工作。

### 巡检控制

#### 巡检状态

- **未启动**：灰色状态，显示"未启动"
- **巡检中**：绿色状态，显示"巡检中"，标题旁有绿色闪烁指示灯

#### 控制按钮

| 按钮 | 状态 | 功能 |
|------|------|------|
| 开始巡检 | 未启动时显示 | 进入巡检模式 |
| 退出巡检 | 巡检中显示 | 退出巡检模式并恢复正常监测 |

> ⚠️ **注意**：巡检模式下会暂停正常的报警逻辑，仅用于功能测试。

### 灶台操作

巡检模式下，可对每个灶台进行以下操作：

| 按钮 | 图标 | 功能 |
|------|------|------|
| 离人检测 | 👤 | 测试人形检测功能，播报当前是否检测到人 |
| 动火检测 | 🔥 | 测试动火检测功能，播报当前是否检测到动火 |
| 报警演示 | 🔔 | 完整演示报警流程（预警→报警→切电） |
| 切电 | ⚡ | 直接测试切电功能 |

### 全局强制动作

对所有灶台同时执行操作：

| 按钮 | 功能 |
|------|------|
| 预警 | 触发所有灶台的预警状态和语音播报 |
| 报警 | 触发所有灶台的报警状态和语音播报 |
| 切电 | 触发所有灶台的切电动作和语音播报 |

### 执行进度

执行操作时显示：
- 进度条
- 当前执行步骤名称
- 详细信息

### 巡检结果

以列表形式显示所有巡检操作的历史记录：
- **成功**：绿色边框，显示成功信息
- **警告**：黄色边框，显示警告信息
- **错误**：红色边框，显示错误信息

每条记录包含：
- 灶台名称或操作类型
- 操作结果详情
- 执行时间（时:分:秒）

## 系统设置

系统设置页面包含多个配置模块。

### 网络状态

显示当前网络连接信息：

| 项目 | 说明 |
|------|------|
| 网络类型 | WiFi / 以太网 |
| IP地址 | 当前设备IP |

点击右上角刷新按钮可手动刷新网络状态。

### 远程服务器配置

配置与远程管理平台的连接：

#### 连接状态

- **已连接**：绿色，显示"已连接"
- **连接中**：黄色，显示加载动画
- **连接失败**：红色，显示错误信息

#### 配置项

| 字段 | 说明 |
|------|------|
| 启用远程连接 | 开关控制是否连接远程服务器 |
| 服务器地址 | 远程服务器URL，如 `https://vis.example.com` |
| WebSocket路径 | WebSocket端点路径，默认 `dhlr/socket` |
| 登录接口 | 登录API路径，默认 `/login` |
| 用户名 | 远程平台账号 |
| 密码 | 远程平台密码（已保存会显示占位符） |

#### 操作按钮

- **校验登录**：测试账号密码是否正确，成功后自动保存凭证

### 串口配置

配置与电流检测模块的串口通信：

#### 状态指示

- **已连接**：绿色，串口正常打开
- **未连接**：红色，串口未打开或打开失败

#### 配置项

| 字段 | 说明 |
|------|------|
| 启用串口 | 开关控制是否启用串口通信 |
| 串口路径 | 下拉选择可用串口设备 |
| 波特率 | 选择通信波特率（9600/19200/38400/115200） |
| 轮询间隔 | 电流值查询间隔（秒） |
| 16进制调试日志 | 开启后在日志中打印串口收发数据 |

点击刷新按钮可重新扫描可用串口。

### LoRa配置

## 常见问题

### Q: 页面显示"暂无启用的灶台"怎么办？

A: 需要先在"摄像头管理"中添加摄像头，然后在"灶台配置"中创建灶台并设置ROI区域。

### Q: 摄像头显示离线怎么办？

A: 
1. 检查摄像头是否正常通电和连接
2. USB摄像头确认设备号是否正确
3. RTSP摄像头检查网络连接和认证信息

### Q: 串口显示未连接怎么办？

A:
1. 检查串口线是否正确连接
2. 确认选择了正确的串口设备
3. 确认波特率设置与硬件一致
4. 重启系统后重试

### Q: 远程服务器连接失败怎么办？

A:
1. 确认服务器地址正确（包含协议头 http/https）
2. 使用"校验登录"功能测试账号密码
3. 检查网络连接是否正常
4. 查看错误信息获取详细失败原因
